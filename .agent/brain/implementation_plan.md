# Implementation Plan: Dual Testing Strategy (PHPUnit + Pest)

## Objective
To seamlessly support both legacy PHPUnit (`Tests\*`) and modern Pest (`Pest\*`) test suites within the CMSForNerd v3.5 environment, ensuring 100% backward compatibility while enabling future migration.

## Proposed Changes

### 1. Dual Test Execution Setup
Pest PHP is built *on top* of PHPUnit. By default, running Pest will also discover and execute native PHPUnit tests. However, we want strict, isolated pipelines to guarantee zero cross-contamination and allow explicit targeted testing during the transition phase.

#### [MODIFY] [composer.json](file:///d:/Users/LinuxMalaysia/CMSForNerd-Project/CmsForNerd/composer.json)
- **Action**: Update the `scripts` block to establish dedicated testing commands:
    * `"test:phpunit"`: `phpunit --colors=always` (Legacy native executor)
    * `"test:pest"`: `pest --colors=always` (Modern Pest executor - runs both)
    * Change the main `"test"` script to run *both* sequentially or default to `"test:pest"` which natively supports executing standard PHPUnit test cases.
- **Reason**: Simplifies execution while giving developers explicit control over which engine they are invoking.
